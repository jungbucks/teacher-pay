# 교사 수당 계산기 — 설계 문서 (2026-07-18)

> 호봉·근무연수만 넣으면 정근수당·정근수당 가산금·명절휴가비가 즉시 뜨는 공개 계산기. 새 폴더 `teacher-pay`, GitHub Pages 배포.
> 브레인스토밍 확정: 3수당 · 호봉 선택→봉급표 자동 · 전체 교사 공개 · UI가 승부처(즉시 반응).

## 1. 목적·훅

- 단일 임무: "매번 블로그로 포스팅하던 수당 안내"를 링크 하나로 대체. 호봉 고르는 순간 옆에 결과가 살아있게 뜬다.
- 훅: **"내 정근수당, 이번엔 얼마?"** — 호봉 드롭다운을 건드리는 순간(첫 조작 3초) 결과 카드가 즉시 갱신. 급여명세서라는 교사의 일상 사물에서 출발.
- UI 원칙(사용자 강조): 유사 서비스가 많으므로 **즉시성·명료함·모바일**에 최대한 공들인다. 입력 옆에 결과가 실시간으로.

## 2. 입력·출력

- **입력**: ① 호봉(1~40 드롭다운/슬라이더) → 봉급표에서 월봉급액 자동 표시 ② 근무연수(숫자)
- **출력 카드 4개**(입력 즉시 live 갱신):
  1. **정근수당** = 월봉급액 × 근무연수별 지급률 — **1월·7월 각각** 지급
  2. **정근수당 가산금** = 근무연수 구간별 월 정액 — **매월**
  3. **명절휴가비** = 월봉급액 × 60% — **설·추석 각각**
  4. **연간 합계** = 정근수당×2 + 가산금×12 + 명절휴가비×2
- 각 카드에 산식을 함께 노출(교육적·검증 가능). 금액은 천단위 콤마.

## 3. 데이터 (숫자는 지어내지 않음 — 출처 명시)

세 데이터를 로직에서 분리:
- `data/salary-2026.js` — 2026 교원 봉급표 호봉 1~40 월봉급액. **출처: 인사혁신처 2026 공무원봉급표(mpm.go.kr), 복수 앵커 교차검증 완료.**
- `data/allowance-rules.js` — 정근수당 지급률표 + 가산금 구간표 + 명절휴가비율(60%).
- `js/calc.js` — 순수 계산 함수(데이터/로직 분리, verify가 단위 검산).

## 4. 정확도 관리 (최대 리스크)

- **verify 자동 검산**: 봉급표 40행·단조증가·앵커값(9호봉 2,495,600 / 40호봉 6,205,700) 일치 / 지급률 11구간 0~50% / 계산 함수 예시(월봉급 3,000,000·근무10년 → 정근수당 1,500,000, 명절비 1,800,000, 가산금 60,000 / 근무 미만 경계값).
- **교사 검수 관문(검수 전 배포 금지)**: `docs .../검증표`에 각 데이터의 출처·확신도 기록.
  - 봉급표: 高(공식 교차검증) / 지급률: 高(장기 고정) / **가산금: 中 — 블로그 출처 상충, 선생님이 공식 별표2와 대조 필수** / 명절 60%: 高.
- 하단 고지: "2026년 기준. 개인 상황(중도 임용·감봉·근무연수 산입 예외 등)에 따라 실제 지급액과 다를 수 있는 참고용입니다." + 출처 링크(인사혁신처·국가법령정보센터).
- 근무연수는 사용자 입력 — 군경력 산입 등 예외는 자동 계산하지 않고 "근무연수 산정은 규정 확인" 안내.

## 5. 기술·디자인·배포

- 바닐라 HTML/CSS/JS, 빌드 없음. classic script(전역) 또는 단일 module — file:// 미리보기 가능하게 classic 권장.
- **frontend-design 적용**(사용자 지시): UI가 승부처. 급여명세서/영수증 모티프 후보, 입력↔결과 실시간, 모바일 우선, 큰 숫자 가독성.
- 검증: `test/verify.mjs`(봉급표·계산 예시) + `test/probe.html`(호봉 변경 즉시 갱신·4카드·연합계·근무연수 경계).
- 배포: 새 GitHub Pages 레포(`teacher-pay`) — 레포 생성·푸시는 사용자 몫.
